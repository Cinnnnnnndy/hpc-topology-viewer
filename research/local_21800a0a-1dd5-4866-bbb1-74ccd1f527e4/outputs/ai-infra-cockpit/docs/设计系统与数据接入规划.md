# 设计系统与真实数据接入规划

> 面向下一阶段工程化（Claude Code 深入开发）的两份施工图：①页面整体设计系统 ②伪遥测 → 真实数据替换。
> 现状基线：`index.html`（v17，单文件，样式在 `:root` CSS 变量 + 内联，数据为确定性伪遥测）。

---

## 一、设计系统（Design System）

### 1.1 设计令牌（Token）——从现有 CSS 变量升格

现有 `:root` 已是雏形，工程化时按三层拆分：

```
原始层 primitives:  gray-900 #0d1117 … blue-400 #58a6ff …（GitHub Dark 系）
语义层 semantic:    --bg / --panel / --border / --txt / --dim
                    --ok #3fb950  --warn #d29922  --danger #f85149  --info #58a6ff
维度层 domain:      --dim-tp #39c5cf  --dim-pp #d29922  --dim-dp #58a6ff  --dim-ep #a371f7
                    --dim-sp #2ea27a  --dim-job-train/--dim-job-infer
                    --hw-util(明度梯度) --hw-mem(#39c5cf→#d29922→#f85149) --hw-link(绿/橙/红)
```

**硬性约束（来自交互文档第 12 章，必须编码进 lint/review）**：
- 维度色（TP/PP/DP/EP…）只用于"软件分组"，跨所有视图（左画布/3D/矩阵/泳道/连线/徽章）同一维度同一色相；
- 硬件测量只允许调制明度/透明度/形变/线宽，不得占用新色相；红色保留给"异常/偏差"，绿色保留给"健康/已掩盖"。
- 主题：暗色为主（NOC 值班场景）；预留亮色主题（汇报场景，参考用户光版截图），实现方式=仅替换原始层。

### 1.2 组件清单（现有内联样式 → 组件化）

| 组件 | 现状 | 组件化要点 |
|---|---|---|
| 按钮组（透镜/连线/形态/排列/关系/剧本） | class 变体 `.lensbtn .on` | 统一 `<ToggleGroup>`：互斥/多选两种模式、维度色 on 态 |
| 层级行 lvrow | HTML 模板 | `<LevelRow>`：标题/计数/KPI 带/body 插槽、focus/ghost 态 |
| 徽章 badge | `.badge .live` | `<CommBadge>` 维度色 + live 发光 |
| 面板（filterbar/topn/probe/HUD/memPanel） | 各自内联 | `<Panel>` + `<KV>`；Probe/HUD 内实体词一律 `<EntityLink>`（可点，路线图"全文本可点"） |
| 画布组件（cvL2/3/4、mmcv、泳道/矩阵/基线/sparks） | 手写 canvas | 统一 `CanvasView` 基类：fitCanvas(DPR)、pick、hover、resize 订阅 |
| 3D 场景 | 单文件函数群 | 拆 `scene/`：entities(实例网格)、layouts(pos 函数=形态)、lenses(着色)、wires(连线)、camera、indicators |
| 时间轴/抽屉/场景剧本栏 | 内联 | `<PhaseTrack>` `<ModeBDock>` `<ScenarioBar>` |

### 1.3 布局与交互规范

- 三分屏栅格：左 430px 固定 + 右自适应 + 底部 Dock（248px，挤压式）；断点 <1280px 时左栏可折叠为图标条。
- 反馈三原则：任何选择必须有 ①就地高亮 ②计数/面包屑变化 ③（跨视图时）对侧联动；相机移动永远伴随非相机反馈（v6 教训）。
- 文案规范：实体名 = `类型+编号`（Host 320 / 柜40 / SN1 / G7 / PP9·TP3），全部可点；解释性文案跟随维度色。

### 1.4 工程化路径（建议顺序）

1. 保持单文件可运行的前提下，先把 `<script>` 按现有分节拆为 ES modules（Vite + TS），`S` 状态机改为带订阅的 store（保留"单一状态源"不变量）；
2. 抽 token 到 `tokens.css`；组件按 1.2 清单渐进抽取（不引重框架，Lit/Preact 或原生 Web Components 均可，保留 canvas/three 手绘内核）；
3. `tools/harness.js` 升级为 Playwright（真浏览器 + WebGL 截图对比），保留 jsdom 版做快速状态机断言。

## 二、真实数据替换（Data Adapter）

### 2.1 原则：伪遥测唯一出口 → DataSource 接口

现在所有伪数据集中在"数据模型"分节（health/expertHosts/anomExperts/memLevel/JOBS/laneLayout/renderBw 曲线/renderMx vol）。
替换方式：**定义 DataSource 接口，MockSource 实现现状，RealSource 对接真实链路**，渲染层零改动。

```ts
interface DataSource {
  // 静态/低频 —— 启动或作业变更时拉取
  getTopology(): Topology;            // pods/cabinets/hosts/chips + 物理位置 + 链路表
  getJobMap(): JobMap;                // host → {job, framework, 并行策略(TP/PP/DP/EP 尺寸), 整网层数}
  getModelMapping(): ModelMapping;    // layer → {ppStage, isMoE, expertIds} ; expert → host

  // 高频遥测 —— 秒级轮询/推送（模式 A 热力大盘）
  subscribeTelemetry(cb: (t: TelemetryFrame) => void): Unsub;
  // TelemetryFrame: per-host {util, memUsed, temp, power, healthFlags}
  //                 per-link {bw, congestion, status: up|degraded|down}
  //                 global   {mfu, stepTime, loss, lr, lossScale, alerts[]}

  // 深度数据 —— 按需拉取（模式 B / 触发式抓拍）
  getTrace(hostId, stepRange): LaneTrace;        // 双轨事件[{stream, op, ts, dur, commGroup}]
  getCommMatrix(stepRange, groupBy): Matrix;     // rank×rank 或 组×组 字节数
  getBusbwBaseline(collective): CurveWithBand;   // 常态化 hccl_test 采样 + 历史带
  getOverlapByRank(stepRange): number[];         // HTA 式逐 rank overlap%
}
```

### 2.2 数据源对接映射（昇腾栈优先，NV 栈等价物备注）

| 驾驶舱消费点 | 昇腾侧来源 | 等价物 |
|---|---|---|
| Topology/链路 | 集群 CMDB + 灵衢拓扑接口 | NVML topo / DCGM |
| util/mem/temp/power | NPU-SMI / DCMI 采集 → Prometheus | DCGM-Exporter |
| MFU/stepTime/loss/lr | 训练框架回调（MindSpeed hooks）→ 时序库 | 同（Megatron callbacks） |
| 泳道 Trace | CANN Profiling / msprof → MindStudio Insight 同款 trace_view.json（工具已支持自动遍历多卡文件） | PyTorch Profiler(Kineto) json |
| 通信矩阵/通信域 | HCCL 日志解析（通信域、算法、每 hop NIC 归属）| NCCL_DEBUG=INFO Channel 解析 |
| busbw 基线 | hccl_test 定时任务 | nccl-tests |
| overlap%/straggler | 对 trace 做 HTA 同类分析（可直接复用 HTA 的 DataFrame 口径） | HTA |
| 告警/触发式抓拍 | 监控告警规则 → 触发 N-step profiling 开关 | 同 |

### 2.3 分级加载与规模策略（万卡前置设计）

- **三级数据节奏**：静态拓扑（启动一次）/ 遥测帧（1-5s，仅可视区域+聚合值）/ 深度数据（点击才拉）。
- **聚合下推**：L4 以上只传柜级/Pod 级聚合（典型/红占比/峰——KPI 带的口径即接口口径）；下钻到 L3/L2 才拉单体明细。前端 LOD 与数据 LOD 同构。
- **回放**：遥测帧落环形缓冲（如 30min），时间轴 Scrubber 在模式 A 回放帧、在模式 B 回放 trace，两套时间源在 UI 上用同一播放头。
- **确定性伪源保留**：MockSource 永久保留（演示/测试/离线开发），harness 断言全部跑在 Mock 上。

### 2.4 替换施工顺序

1. 抽 `MockSource`（把现有伪数据函数原样搬入，渲染层改为消费接口）——纯重构，harness 全绿为准；
2. 接遥测帧（Prometheus HTTP API 轮询 → TelemetryFrame），健康度/显存/任务透镜变真；
3. 接 trace_view.json 解析 → 泳道/计数器轨/overlap 变真（先离线文件，后触发式抓拍链路）；
4. 接 HCCL 日志 + hccl_test → 通信矩阵/连线镜头/busbw 变真；
5. 拓扑接口 + 多 Pod 数据 → 打开 SN1-63 的下钻（当前仅 SN0 有细节）。
